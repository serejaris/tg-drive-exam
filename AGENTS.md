# Agent Rules — tg-drive-exam sandbox

Это песочница для тестирования агентов на задаче «Telegram miniapp для теоретического экзамена на права CABA (Аргентина), класс B».

Тестовые данные лежат в `/Users/ris/Downloads/caba_licencia_miniapp_pack_v1/data/` (40 вопросов в ES+RU, правила теста, источники). Эти файлы — **read-only** для агента: не модифицировать, не перемещать, только читать.

## Главное правило: каждый проект — в новой папке

**Каждая попытка реализации (каждый «проект», который делает агент) должна создаваться в новой подпапке** внутри `tg-drive-exam/`, а не в корне и не поверх существующей попытки.

### Как именовать папку

Формат: `NNN-<slug>/`, где
- `NNN` — трёхзначный порядковый номер попытки (`001`, `002`, `003`, …)
- `<slug>` — короткий kebab-case-идентификатор стека или подхода: `001-next-app`, `002-vanilla-webapp`, `003-vue-pinia`, `004-telegram-miniapp-react`

Номер = `max(existing) + 1`. Перед созданием папки агент обязан проверить, какие `NNN-*` уже есть, чтобы не столкнуться.

### Что внутри каждой папки

Каждая папка-проект должна быть **самодостаточной**:
- свой `README.md` (что за подход, как запускать, какие допущения)
- свой `package.json` / `pyproject.toml` / аналог, со своими зависимостями
- свой код, свои конфиги, свои билды
- никаких симлинков на соседние папки и никакого импорта кода из соседних попыток

Если нужно переиспользовать решение — скопировать файлы явно, а не ссылаться наружу.

### Что запрещено

- ❌ Создавать файлы проекта в корне `tg-drive-exam/` (здесь только `AGENTS.md`, этот файл, и служебные — ничего проектного)
- ❌ Модифицировать чужую папку-попытку (`001-*` не трогает `002-*` и наоборот)
- ❌ Перезаписывать существующую `NNN-*` — если хочется «переделать», создаётся новая `NNN-<slug>-v2/` или просто следующий номер
- ❌ Трогать исходные данные в `~/Downloads/caba_licencia_miniapp_pack_v1/` — они read-only
- ❌ Использовать общий `node_modules` / `.venv` на корне — только внутри своей папки

### Работа с данными

Данные копировать или читать по абсолютному пути:
- Копия: `cp ~/Downloads/caba_licencia_miniapp_pack_v1/data/*.json NNN-<slug>/data/`
- Чтение по пути: fetch/readFile с абсолютным путём к `~/Downloads/.../data/questions.caba.b.starter40.es-ru.json`

Для UI брать двуязычные файлы (`*.es-ru.json`), не оригиналы — RU-поля уже есть.

### Что фиксировать при завершении попытки

В `NNN-<slug>/README.md` коротко:
- Стек (framework, язык, ключевые библиотеки)
- Как запустить локально (одна команда)
- Что сделано / что не доделано
- Известные проблемы

Это нужно, чтобы при сравнении попыток было понятно, что агент успел и где застрял.

## Почему так

Мы сравниваем разные агентские подходы на одной задаче. Изоляция по папкам даёт:
1. Чистое сравнение — никто не «помогает» другой попытке, случайно не ломает её
2. Возможность откатиться — удалить одну папку, не задев остальные
3. Прозрачную историю — `ls` в корне сразу показывает все попытки
